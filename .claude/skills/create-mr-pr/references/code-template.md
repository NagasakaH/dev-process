# Codeモード テンプレート

コードレビュー用MR/PRのdescriptionテンプレート。

## テンプレート構造

```markdown
## {タイトル}

### 概要

{変更の目的・背景・スコープの簡潔な説明}

### 変更内容

{主要な変更点のリスト}

### AI自動チェック

> ⚠️ **根拠記載ルール**:
> - **`- [x]` 項目（チェック済み）**: 根拠の記載が必須
> - **`- [ ]` 項目（対象外）**: 対象外の理由を項目テキストに `（対象外: {理由}）` の形式で必ず付記
>
> 根拠には以下のいずれか1つ以上を含めること:
> - **テスト結果**: 実行コマンドと出力（例: `npm test → 247 tests passed`）
> - **検証方法**: 確認に使用した手順やツール（例: `git diff --stat で変更範囲を確認`）
> - **確認した証拠**: ログ、スクリーンショット、CI結果URL等の客観的証拠
>
> ✅ 有効な根拠: `npm test → 247テスト全通過 (3.2s)` / `eslint src/ → 0 errors` / `CI #1234 passed`
> ❌ 無効な根拠: `OK` / `確認済み` / `テスト実行済み` / `問題なし`（具体的コマンド・結果・証拠がない）
>
> **対象外の例**:
> - `- [ ] テスト全パス（対象外: ドキュメントのみの変更のためテスト対象なし）`
> - `- [ ] E2Eテスト（対象外: テスト戦略にE2Eが含まれていない）`

- [ ] テスト全パス
  - 根拠: {テスト実行コマンドと結果を記載}
- [ ] ビルド成功
  - 根拠: {ビルドコマンドと結果を記載}
- [ ] リント通過
  - 根拠: {リントコマンドと結果を記載}
- [ ] CI/パイプライン成功
  - 根拠: {CI実行結果URLまたはステータスを記載}
- [ ] カバレッジ低下なし
  - 根拠: {カバレッジ比較結果を記載}
- [ ] シークレット/認証情報の混入なし
  - 根拠: {検出ツール実行結果またはgrep結果を記載}
- [ ] マージコンフリクトなし
  - 根拠: {ベースブランチとのマージ確認結果を記載}

### AI+人間チェック

- [ ] Acceptance criteria充足（根拠付き）
- [ ] 破壊的変更なし（or 文書化済み）
- [ ] エラーハンドリング適切
- [ ] セキュリティ考慮
- [ ] パフォーマンス影響考慮
- [ ] ドキュメント更新済み
- [ ] コミットメッセージ明確
- [ ] 修正範囲がDR合意済みリポジトリに限定
```

## テンプレート生成ルール

1. **概要**: コミットメッセージやブランチ名から要約を自動生成。
2. **変更内容**: `git diff --stat` の結果をもとにリスト化。
3. **チェックボックス**: 全て未チェック状態で作成。
4. **AC根拠**: code-review後にAIが各ACに対する根拠（テスト結果等）を記入。
5. **AI自動チェック根拠**: チェック済み（`- [x]`）項目は根拠の記載が必須。未チェック（`- [ ]`）項目で対象外の場合は対象外理由の記載が必須。根拠には具体的なコマンド名・実行結果・数値・URL等の客観的証拠を含めること。「OK」「確認済み」等の抽象的記述のみの根拠は構造検証で却下される。

## Code MR/PR完了後の運用

- code-reviewスキルがレビュー結果をコメントとして投稿
- AIが自動チェック項目を検証し、descriptionを更新
- 全指摘解消後、draftを解除
- 人間がpr_reviewチェックポイントでレビュー
- 承認後、merge実施
