# Codeモード テンプレート

コードレビュー用MR/PRのdescriptionテンプレート。

## テンプレート構造

```markdown
## {タイトル}

### 概要

{変更の目的・背景・スコープの簡潔な説明}

### 変更内容

{主要な変更点のリスト}

### AI自動チェック

- [ ] テスト全パス
- [ ] ビルド成功
- [ ] リント通過
- [ ] CI/パイプライン成功
- [ ] カバレッジ低下なし
- [ ] シークレット/認証情報の混入なし
- [ ] マージコンフリクトなし

### AI+人間チェック

- [ ] Acceptance criteria充足（根拠付き）
- [ ] 破壊的変更なし（or 文書化済み）
- [ ] エラーハンドリング適切
- [ ] セキュリティ考慮
- [ ] パフォーマンス影響考慮
- [ ] ドキュメント更新済み
- [ ] コミットメッセージ明確
- [ ] 修正範囲がDR合意済みリポジトリに限定
```

## テンプレート生成ルール

1. **概要**: コミットメッセージやブランチ名から要約を自動生成。
2. **変更内容**: `git diff --stat` の結果をもとにリスト化。
3. **チェックボックス**: 全て未チェック状態で作成。
4. **AC根拠**: code-review後にAIが各ACに対する根拠（テスト結果等）を記入。

## Code MR/PR完了後の運用

- code-reviewスキルがレビュー結果をコメントとして投稿
- AIが自動チェック項目を検証し、descriptionを更新
- 全指摘解消後、draftを解除
- 人間がpr_reviewチェックポイントでレビュー
- 承認後、merge実施
