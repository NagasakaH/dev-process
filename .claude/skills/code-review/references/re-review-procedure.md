# 再レビュー手順

## 再レビュー時の処理（round > 1）

再レビューでは以下の追加手順を実行します：

1. **前ラウンドのグループ構成を確認**: 意図グループが変化していないかチェック（新コミット追加時は再分析）
2. **グループ別に前ラウンドの指摘確認**: 各グループの未解決指摘を優先チェック
3. **解決確認**: 対応済み指摘のステータスを resolved に更新
4. **新規指摘追加**: 新たに発見された問題を新しい ID で追加
5. **デュアルモデルレビュー**: 再レビューでも Opus 4.6 + GPT-5.5 の並列レビューを実施
6. **統合サマリーに前ラウンドとの差分を記載**

## ラウンド管理ルール

- **初回レビュー**: round 1 で開始
- **再レビュー**: 前ラウンドの round をインクリメント
- **指摘事項**: 全ラウンドの指摘を累積保持（解決ラウンドを追跡）
- **グループ構成**: 新コミットがなければ前ラウンドと同じグループ構成を維持
- **ラウンド記録**: 各ラウンドの結果サマリーを累積記録

## 統合サマリーへの追記（round > 1）

```markdown
## 前ラウンドからの変化
- 解決済み: {count}件（グループ別内訳: G1={n}, G2={n}）
- 新規指摘: {count}件
- 未解決: {count}件
```
