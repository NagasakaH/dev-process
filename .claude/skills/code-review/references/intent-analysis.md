# 意図分析・グループ化

## コミット一覧取得

```bash
BASE_SHA=$(git merge-base HEAD origin/main)
HEAD_SHA=$(git rev-parse HEAD)
git log --oneline "$BASE_SHA..$HEAD_SHA"
```

## 意図分析プロセス

各コミットについて以下を分析:

1. **コミットメッセージ** — プレフィックス（feat/fix/refactor/docs/test/chore）と説明
2. **変更ファイル一覧** — `git diff --name-only <SHA>^..<SHA>`
3. **diff概要** — `git diff --stat <SHA>^..<SHA>`

## 意図カテゴリ

| カテゴリ | 判定基準 |
|----------|----------|
| feature | 新規機能追加（feat: プレフィックス、新ファイル作成） |
| bugfix | バグ修正（fix: プレフィックス、既存ファイルの修正） |
| refactor | リファクタリング（refactor: プレフィックス、テスト変更なし） |
| test | テスト追加・修正（test: プレフィックス、テストファイルのみ変更） |
| docs | ドキュメント変更（docs: プレフィックス、.md/.txt のみ） |
| config | 設定変更（chore/ci: プレフィックス、設定ファイルのみ） |
| mixed | 上記の複合（1コミットに複数意図が混在） |

## グループ化ルール

1. **同一カテゴリのコミットをグループ化**
2. **関連ファイルを変更するコミットは同グループ** — 同じモジュール/パッケージを触るコミット
3. **mixed コミットは最も影響の大きいカテゴリに分類**（判断が難しい場合は独立グループ）
4. **グループが1つの場合もそのまま1グループとして処理**

## 出力形式

```
グループ1: [カテゴリ名] — [概要説明]
  - コミットSHA1: メッセージ
  - コミットSHA2: メッセージ
  対象ファイル: file1.ts, file2.ts, ...

グループ2: [カテゴリ名] — [概要説明]
  - コミットSHA3: メッセージ
  対象ファイル: file3.py, ...
```

## 注意事項

- グループ数が多すぎる場合（5以上）は、関連性の高いグループの統合を検討
- 1コミットのみのブランチでも意図分析は実施（グループは1つ）
- 意図分析結果は統合サマリーの冒頭に記載する
