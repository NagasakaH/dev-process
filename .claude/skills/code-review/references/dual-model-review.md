# デュアルモデルレビュー

## 概要

各意図グループに対し、2つのモデルが並列でレビューを実施し、第3のモデルが統合・判定を行う3フェーズ方式。

## フェーズ構成

```
フェーズ1: 並列レビュー
├── Opus 4.6 → レビュー結果A
└── Codex 5.3 → レビュー結果B

フェーズ2: 統合判定（Opus 4.6）
├── 結果A + 結果B を入力
├── 重複指摘の統合
├── 片方のみの指摘の妥当性判定
└── 統合レビュー結果を出力

フェーズ3: レポート生成
└── round-NN-group-MM.md 出力
```

## フェーズ1: 並列レビュー

各モデルに以下を入力して**並列実行**（task ツールの background モード）:

### レビュアーへの指示

```
以下の差分をレビューしてください。

## 意図グループ
- グループ名: {group_name}
- 対象コミット: {commit_list}

## 差分
{git diff output}

## MR/PRレビュー要求項目（存在する場合）
{extracted_review_items}

## 設計ドキュメント（存在する場合）
{design_docs}

## チェックリスト
8カテゴリ + MR要求項目（該当する場合）のチェックリストに基づきレビュー。

各指摘について:
- ID: CR-NNN
- 重大度: 🔴Critical / 🟠Major / 🟡Minor / 🔵Info
- カテゴリ: チェックリストのカテゴリ番号
- 該当ファイル: パス:行番号
- 問題の説明
- 修正提案（具体的なコード例）
```

### モデル指定

| 役割 | モデル | task ツールの model パラメータ |
|------|--------|-------------------------------|
| レビュアーA | Claude Opus 4.6 | `claude-opus-4.6` |
| レビュアーB | GPT Codex 5.3 | `gpt-5.3-codex` |

## フェーズ2: 統合判定

Opus 4.6（統合担当）に両レビュー結果を入力:

```
以下の2つのレビュー結果を統合してください。

## レビュー結果A（Opus 4.6）
{review_a}

## レビュー結果B（Codex 5.3）
{review_b}

## 統合ルール
1. 両モデルが同じ箇所を指摘 → 統合し、より詳細な方の説明を採用
2. 片方のみが指摘 → 妥当性を判定し、採用/棄却を決定（理由を明記）
3. 重大度が異なる → より高い方を採用（安全側に倒す）
4. 修正提案が異なる → より適切な方を採用、または両方を提示
5. 総合判定は最も厳しい判定を採用

統合後の出力形式:
- 統合指摘一覧（ID、重大度、カテゴリ、ファイル、説明、修正提案、出典[A/B/Both]）
- 棄却された指摘一覧（理由付き）
- 総合判定と理由
```

## モデル指定の注意

- task ツールの `model` パラメータで明示的にモデルを指定する
- レビュアーBは `gpt-5.3-codex` を使用（GitHub Copilot CLI で利用可能）
- 統合担当は `claude-opus-4.6` を使用
