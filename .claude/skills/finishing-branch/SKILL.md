# ブランチ完了スキル

開発作業の完了を明確な選択肢の提示と選択されたワークフローの実行でガイドします。

## 入力

- **ブランチ名**: 現在の作業ブランチ（`git branch --show-current` で取得）
- **チケットID**（任意）: ブランチ名やコミットメッセージから推測、またはユーザーから提供
- **ベースブランチ**: マージ先ブランチ（`git merge-base` 等で特定）

## 主要機能

### プロセス

1. **テスト検証**: テストが通ることを確認
2. **ベースブランチ決定**: main/masterから分岐したか確認
3. **オプション提示**: 4つの選択肢を提示
4. **選択実行**: 選択されたワークフローを実行
5. **Worktreeクリーンアップ**: 必要に応じて削除

### 4つの選択肢

```
実装完了。次に何をしますか？

1. ローカルで <base-branch> にマージ
2. プッシュしてPRを作成
3. ブランチをそのまま保持（後で対応）
4. この作業を破棄
```

## クイックリファレンス

| オプション        | マージ | プッシュ | Worktree保持 | ブランチ削除 |
| ----------------- | ------ | -------- | ------------ | ------------ |
| 1. ローカルマージ | ✓      | -        | -            | ✓            |
| 2. PR作成         | -      | ✓        | ✓            | -            |
| 3. そのまま       | -      | -        | ✓            | -            |
| 4. 破棄           | -      | -        | -            | ✓（強制）    |

## 各オプションの実行手順

### 1. ローカルマージ

```bash
git checkout <base-branch>
git merge --no-ff <feature-branch>
# マージ後にテスト再実行で確認
git branch -d <feature-branch>
```

### 2. PR作成

```bash
git push -u origin <feature-branch>
gh pr create --base <base-branch> --title "<タイトル>" --body "<説明>"
```

- チケットIDがあればPRタイトルや本文に含める
- PRテンプレートがリポジトリにあれば使用する

### 3. ブランチ保持

- 特に操作なし。ブランチとworktreeをそのまま残す

### 4. 作業破棄

```bash
# ユーザーの明示的な確認後のみ実行
git checkout <base-branch>
git branch -D <feature-branch>
```

## レッドフラグ

**絶対にしない:**
- テスト失敗のまま進む
- マージ結果のテスト検証なし
- 確認なしで作業削除

## 使用タイミング

- 実装完了後
- 全テスト通過後
- 作業統合方法の決定時

## 完了報告

選択したアクションの実行結果をユーザーに報告する：

| アクション | 報告内容                         |
| ---------- | -------------------------------- |
| `merge`    | マージコミットSHA、ベースブランチ |
| `pr`       | PR URL                          |
| `keep`     | ブランチ名、現在の状態           |
| `discard`  | 削除されたブランチ名             |
