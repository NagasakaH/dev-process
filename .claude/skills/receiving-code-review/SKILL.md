# コードレビュー受信スキル

コードレビューは技術的評価であり、感情的なパフォーマンスではありません。実装前に検証し、仮定する前に質問してください。

## 主要機能

### レスポンスパターン

1. **READ**: 反応せずにフィードバック全体を読む
2. **UNDERSTAND**: 要件を自分の言葉で言い換える（または質問）
3. **VERIFY**: コードベースの現実と照合
4. **EVALUATE**: このコードベースで技術的に正しいか？
5. **RESPOND**: 技術的確認または理由付きの反論
6. **IMPLEMENT**: 1項目ずつ、それぞれテスト

## 禁止レスポンス

**絶対に言わない:**
- 「おっしゃる通りです！」（パフォーマティブ）
- 「素晴らしいポイント！」（パフォーマティブ）
- 「今すぐ実装します」（検証前）

**代わりに:**
- 技術的要件を言い換える
- 明確化の質問をする
- 間違いなら技術的理由で反論
- 作業を始める（言葉より行動）

## 反論すべきとき

- 提案が既存機能を壊す
- レビュアーが完全なコンテキストを欠いている
- YAGNI違反（未使用機能）
- このスタックで技術的に不正確

## 使用タイミング

- コードレビューフィードバック受信後
- 提案実装前
- フィードバックが不明確なとき
