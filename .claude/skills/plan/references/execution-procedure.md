# 実行手順・完了レポート・エラーハンドリング

planスキルの詳細な実行手順と成果物フォーマット。

---

## 実行手順

### 1. 入力の確認

- 設計ドキュメントが存在し、設計が完了していることを確認
- 受け入れ条件が定義されていることを確認
- テスト戦略が提供されていれば読み込む

### 2. 設計内容の分析

design/配下の各ファイルを読み込み、タスク化対象を特定：

- 実装方針からタスク粒度を決定
- API設計からエンドポイント実装タスクを抽出
- データ構造設計からモデル実装タスクを抽出
- 処理フロー設計からロジック実装タスクを抽出
- テスト計画からテスト実装タスクを抽出
- **テスト戦略を参照し、E2Eテストがスコープに含まれる場合は E2Eテスト用タスクを必ず生成する**

### 3. タスク分割の実施

1. タスク識別子の生成
2. 依存関係の判定
3. 並列実行グループの特定
4. 推定所要時間の算出

### 4. 各タスクプロンプト生成

各タスクについて `task0X.md` を生成：

- 設計内容を自動抽出
- TDD方針を組み込み
- 完了条件を明確化

### 5. タスク一覧を task-list.md に出力

`docs/{target}/plan/task-list.md` にタスク一覧と依存関係を出力。

### 6. 親エージェント統合管理プロンプト生成

`parent-agent-prompt.md` を生成：

- 全タスク一覧
- 依存関係グラフ
- 実行順序
- worktree管理手順

### 7. コミット

```bash
git add docs/
git commit -m "docs: タスク計画を作成

- docs/{target}/plan/配下にタスクプロンプトを生成
- 親エージェント統合管理プロンプトを生成"
```

```markdown
- 並列実行グループ数: {parallel_group_count}

### 次のステップ

```

## エラーハンドリング

### 設計ドキュメントが見つからない

```
エラー: 設計ドキュメントが見つかりません
ディレクトリ: {design_dir}

設計を完了してから計画を作成してください。
```

### 設計が未完了

```
エラー: 設計が完了していません

設計を完了してから計画を作成してください。
```

### 受け入れ条件が未定義

```
エラー: 受け入れ条件が定義されていません

受け入れ条件を定義してから計画を作成してください。
```
