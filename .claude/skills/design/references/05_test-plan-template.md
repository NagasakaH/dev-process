### 2.4 E2Eテスト実現性チェックリスト

> ⚠️ **必須**: テストスコープにE2Eテストが含まれる場合、以下のチェックリストを**設計フェーズ中に**全て確認すること。
> 不足項目がある場合は**ユーザーに環境準備を依頼**し、準備完了まで設計を先に進めてはならない。

| # | チェック項目 | 確認結果 | 備考 |
|---|-------------|----------|------|
| 1 | Docker / Docker Compose が利用可能か | ☐ 利用可 / ☐ 利用不可 | `docker --version` / `docker compose version` で確認 |
| 2 | 必要なコンテナイメージがプル可能か | ☐ 可能 / ☐ 不可 | レジストリへのアクセス権限を確認 |
| 3 | ネットワーク権限（ポート開放・外部通信）が確保されているか | ☐ 確保済 / ☐ 未確保 | ファイアウォール・プロキシ設定を確認 |
| 4 | 外部サービス（DB, API, メッセージキュー等）へのアクセスが可能か | ☐ 可能 / ☐ 不可 | 接続文字列・認証情報の有無を確認 |
| 5 | E2Eテストフレームワーク・ツールがインストール済みか | ☐ 済 / ☐ 未 | Playwright / Cypress / Selenium 等 |
| 6 | テストデータ・シードデータの準備が可能か | ☐ 可能 / ☐ 不可 | マイグレーション・フィクスチャの有無 |
| 7 | CI/CD環境でのE2E実行が可能か | ☐ 可能 / ☐ 不可 | ランナーのリソース制約を確認 |

#### 不足項目がある場合の対応フロー

```
1. 不足項目を一覧化してユーザーに提示する
2. ユーザーに環境準備を依頼する（具体的な手順を提案）
3. ユーザーから「準備完了」の確認を得るまで設計を中断する
4. 準備完了後、チェックリストを再検証してから設計を続行する
```

> 🚫 **禁止事項**: 「環境が整っていないのでE2Eテストはスキップする」という判断を設計者が独断で行ってはならない。
> E2Eテストのスキップはユーザーの明示的な承認がある場合のみ許容される。
