  - **task12** (弊害検証): coverage 閾値検証・README 階層検証・dry-run revert SHA 特定など追加検証手順により増加。
- **是正**:
  - 全体に 20〜30% のバッファを明記する。
  - task02-02 / task08 / task12 の見積もりを増やす。
