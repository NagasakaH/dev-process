## 目的

Lambda の `JsonOpts` (CamelCase / SnakeCaseLower enum) と一致する Angular 側 TypeScript 型を作成する。`ConfigService` / `TodoApiService` / `TodoComponent` から共有される基底型を提供する。

## 実装ステップ (TDD: RED-GREEN-REFACTOR)

### RED
1. `frontend/src/app/models/todo.spec.ts` (新規) を作成:
   - `const todo: Todo = { id:'x', title:'a', status:'open', createdAt:'2026-01-01T00:00:00Z', updatedAt:'2026-01-01T00:00:00Z' }` が型エラー無くコンパイルできる
   - `'unknown' as TodoStatus` は型エラーになる (`// @ts-expect-error`)
2. `cd frontend && npm run test:unit` (または `tsc --noEmit`) が **FAIL**

### GREEN
3. `frontend/src/app/models/todo.ts` を新規作成:
   ```typescript
   export type TodoStatus = 'open' | 'in_progress' | 'done';
   export interface Todo {
     id: string;
     title: string;
     description?: string;
     status: TodoStatus;
     createdAt: string;
     updatedAt: string;
   }
   export interface TodoCreateRequest { title: string; description?: string; }
   export interface ApiErrorResponse { errors?: string[]; error?: string; }
   export interface UiError { message: string; detail?: string[]; }
   ```
4. `frontend/src/app/services/config.service.ts` のスケルトン (型のみ先出し):
   ```typescript
   export interface AppConfig { apiBaseUrl: string; }
   ```
5. テストが GREEN

### REFACTOR
6. `index.ts` でモデルを barrel export
7. JSDoc コメントで Lambda 由来の制約 (CamelCase / null時省略 / SnakeCaseLower enum) を明記
