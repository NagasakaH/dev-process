## 目的

ランタイム設定 `assets/config.json` を取得し `apiBaseUrl` を保持する `ConfigService` を実装する。`load()` を `APP_INITIALIZER` で起動時に呼び、`apiBaseUrl` が空 / `http(s)://` 非該当時は reject (fail-fast)。

## 実装ステップ (TDD)

### RED
1. `frontend/src/app/services/config.service.spec.ts` を新規作成し下記 UT を追加 (FAIL):
   - **UT-1**: `HttpClientTestingModule` で `/assets/config.json` のリクエストをモックし `apiBaseUrl: 'http://localhost:4566/restapis/x/dev/_user_request_'` を flush → `cfg.apiBaseUrl` が一致
   - **UT-2a**: `apiBaseUrl` 欠落 (`{}`) → `load()` が reject
   - **UT-2b**: `apiBaseUrl` 空文字 → reject
   - **UT-2c**: `apiBaseUrl: 'ftp://...'` → reject
2. `npm run test:unit` で **FAIL**

### GREEN
3. `frontend/src/app/services/config.service.ts` を実装:
   ```typescript
   @Injectable({ providedIn: 'root' })
   export class ConfigService {
     private cfg!: AppConfig;
     constructor(private http: HttpClient) {}
     load(): Promise<void> {
       return firstValueFrom(this.http.get<AppConfig>('/assets/config.json')).then(c => {
         if (!c?.apiBaseUrl || typeof c.apiBaseUrl !== 'string' || !/^https?:\/\//.test(c.apiBaseUrl)) {
           throw new Error('Invalid apiBaseUrl');
         }
         this.cfg = c;
       });
     }
     get apiBaseUrl(): string { return this.cfg.apiBaseUrl; }
   }
   ```
4. `npm run test:unit` で **GREEN**

### REFACTOR
5. エラーメッセージを「設定読み込みエラー」用にユーザフレンドリ化
6. `apiBaseUrl` の末尾スラッシュを除去するヘルパを追加
