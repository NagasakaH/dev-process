## 目的

`compose/docker-compose.yml` の floci `SERVICES` に `s3` を追加し、新規 `nginx` サービス (sidecar) を `floci-net` に追加する。`./frontend/dist` を `:ro` で host volume mount し、ブラウザ配信を nginx 経由の **唯一標準経路** に統一する (RD-001)。

## 実装ステップ (TDD: RED-GREEN-REFACTOR)

### RED
1. `docker compose -f compose/docker-compose.yml config` を取得しベースラインを保存
2. `docker compose up -d nginx` が「サービス未定義」で失敗することを確認

### GREEN
3. `compose/docker-compose.yml`:
   - `services.floci.environment.SERVICES` に `s3` を末尾追加
   - `services.nginx` を追加 (`image: nginx:1.27-alpine`, `ports: ["8080:8080"]`, `networks: [floci-net]`, volumes は `./compose/nginx/default.conf:/etc/nginx/conf.d/default.conf:ro` と `./frontend/dist:/usr/share/nginx/html:ro`, `depends_on: [floci]`)
   - `networks.floci-net` を明示宣言
4. `compose/nginx/default.conf` を新規作成:
   ```nginx
   server {
     listen 8080;
     root /usr/share/nginx/html;
     location / {
       try_files $uri /index.html;
     }
   }
   ```
5. `docker compose up -d floci nginx` が成功し、`docker compose ps` で両者 `running` を確認
6. `curl -I http://localhost:8080/` が 200 もしくは 404 (dist 未配置時) を返すこと、`-X OPTIONS` ではなく静的配信 only であることを確認

### REFACTOR
7. `proxy_pass` 等が含まれていないことを再確認 (API リバースプロキシは禁止)
8. `frontend/dist` がない状態でも nginx コンテナが起動できる (ボリュームマウントは存在しない場合空ディレクトリ扱い)
