## 目的

`src/TodoApi.Lambda/Function.cs` の共通レスポンスヘッダ辞書 (`JsonHeaders`) に CORS ヘッダを追加し、`ApiHandler` の冒頭で `OPTIONS` を 204 + CORS preflight ヘッダで返すハンドラを追加する。MOCK 統合は採用せず Lambda 標準・唯一経路。既存 `TodoApi.UnitTests` を TDD で先に拡張する（RD-006）。

## 実装ステップ (TDD: RED-GREEN-REFACTOR)

### RED
1. `tests/TodoApi.UnitTests/ApiHandlerRoutingTests.cs` (および他関連) の期待値を更新:
   - `POST /todos` 成功 / `BadRequest` / `NotFound` / 500 すべてに `Access-Control-Allow-Origin: *` / `Access-Control-Expose-Headers: Content-Type` を期待
2. 新規 unit case を追加:
   - `OPTIONS /todos` → `StatusCode == 204`, `Access-Control-Allow-Origin: *`, `Access-Control-Allow-Methods: GET, POST, OPTIONS`, `Access-Control-Allow-Headers: Content-Type`, `Access-Control-Max-Age: 600`
   - `OPTIONS /todos/{id}` も同様
3. `dotnet test tests/TodoApi.UnitTests` が **FAIL** することを確認

### GREEN
4. `src/TodoApi.Lambda/Function.cs`:
   - `JsonHeaders` 共通辞書に CORS ヘッダを追加
   - `CorsPreflightHeaders` を別途定義し `Allow-Methods/Headers/Max-Age` を含める
   - `ApiHandler` の入口で `req.HttpMethod == "OPTIONS"` を判定し `204 No Content` + `CorsPreflightHeaders` を返す
5. `dotnet test tests/TodoApi.UnitTests` を **GREEN** で確認

### REFACTOR
6. CORS ヘッダ定義を private static フィールドに集約し重複を排除
7. `dotnet format --verify-no-changes` を実行して整形違反が無いこと
8. 既存 IntegrationTests / E2ETests が壊れていないかローカル smoke 実行
