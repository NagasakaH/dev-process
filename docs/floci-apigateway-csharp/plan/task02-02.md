## 目的

`infra/` に下記を追加する。
- `/todos`, `/todos/{id}` への `OPTIONS` メソッド (`authorization=NONE`) と `aws_api_gateway_integration` (`type=AWS_PROXY` で Lambda 統合)
- APIGW から Lambda OPTIONS 呼び出し用 `aws_lambda_permission`
- フロント成果物配置検証用 `aws_s3_bucket` (名: `frontend`)
- `outputs.tf` に `frontend_bucket` / `frontend_url` (`http://localhost:8080`) 追加

> MOCK 統合 / `aws_s3_bucket_website_configuration` / `aws_api_gateway_method_response.response_parameters` での CORS 宣言は **行わない** (RD-001 / RD-006 / RD-011)。AWS_PROXY 透過に依拠する旨を Terraform コメントで明記。

## 実装ステップ (TDD: RED-GREEN-REFACTOR)

### RED
1. `terraform -chdir=infra validate` を実行し追加リソースが未定義であることを確認
2. ローカルで floci 起動済みの状態で `infra/` の現状の plan を取得しベースラインとする
3. plan 後に下記が含まれていないことを確認 (テスト的 assertion):
   - `aws_api_gateway_method.options_todos`
   - `aws_api_gateway_integration.options_todos_int` (`type=AWS_PROXY`)
   - `aws_s3_bucket.frontend`

### GREEN
4. `infra/main.tf` (もしくは新規 `infra/frontend.tf`) に上記リソースを追加
5. `infra/outputs.tf` に `frontend_bucket` / `frontend_url` を追加
6. `terraform -chdir=infra fmt` / `terraform validate` が成功
7. floci 起動状態で `terraform -chdir=infra plan` がエラー無し
8. `terraform apply` 後に `curl -i -X OPTIONS http://<endpoint>/restapis/.../dev/_user_request_/todos` で `HTTP/1.1 204` + CORS ヘッダを確認 (task02-01 と組み合わせて)

### REFACTOR
9. AWS_PROXY 透過に依拠する旨を Terraform コメントに明記
10. `frontend_url` の用途コメント (nginx ベース URL の組み立てヒント) を追記
11. `aws_s3_bucket.frontend` には実 AWS 流出防止のため tags + コメント (floci ローカル限定) を追加
