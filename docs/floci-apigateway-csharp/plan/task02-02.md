## 目的

`infra/` に下記を追加する。
- `/todos`, `/todos/{id}` への `OPTIONS` メソッド (`authorization=NONE`) と `aws_api_gateway_integration` (`type=AWS_PROXY` で Lambda 統合)
- APIGW から Lambda OPTIONS 呼び出し用 `aws_lambda_permission`
- フロント成果物配置検証用 `aws_s3_bucket` (名: `frontend`)
- `outputs.tf` に `frontend_bucket` / `frontend_url` (`http://localhost:8080`) 追加

> MOCK 統合 / `aws_s3_bucket_website_configuration` / `aws_api_gateway_method_response.response_parameters` での CORS 宣言は **行わない** (RD-001 / RD-006 / RD-011)。AWS_PROXY 透過に依拠する旨を Terraform コメントで明記。

## 実装ステップ (TDD: RED-GREEN-REFACTOR)

### RED — `terraform plan -json` ベースの強い失敗テスト (RP-010)
1. `tests/infra/test-frontend-plan.sh` (新規) を作成し、現状の plan 出力に対して下記を assertion する shell test を実装する。リソース未定義時に **明示的に exit 1** で失敗する:
   ```bash
   #!/usr/bin/env bash
   set -euo pipefail
   terraform -chdir=infra init -backend=false -upgrade >/dev/null
   terraform -chdir=infra validate
   PLAN_JSON=$(terraform -chdir=infra plan -refresh=false -no-color -json 2>/dev/null \
     | jq -s '[.[] | select(.type=="planned_change" or .type=="resource_drift")] | map(.change.resource.addr // .change.resource_addr // .resource.addr) // []')
   # 必須リソース 4 種（OPTIONS×2 メソッド + AWS_PROXY 統合×2 + S3 bucket + lambda_permission×2）
   for addr in \
     'aws_api_gateway_method.options_todos' \
     'aws_api_gateway_method.options_todo_id' \
     'aws_api_gateway_integration.options_todos_int' \
     'aws_api_gateway_integration.options_todo_id_int' \
     'aws_lambda_permission.apigw_options_todos' \
     'aws_lambda_permission.apigw_options_todo_id' \
     'aws_s3_bucket.frontend' ; do
     echo "$PLAN_JSON" | jq -e --arg a "$addr" 'any(. == $a)' >/dev/null \
       || { echo "[FAIL] missing resource in plan: $addr" >&2; exit 1; }
   done
   # AWS_PROXY 統合タイプを確認
   terraform -chdir=infra plan -refresh=false -no-color -json \
     | jq -e 'select(.change.resource.addr=="aws_api_gateway_integration.options_todos_int") | .change.after.type=="AWS_PROXY"' >/dev/null \
     || { echo "[FAIL] options_todos_int is not AWS_PROXY" >&2; exit 1; }
   # MOCK 統合 / response_parameters CORS 宣言が無いことを確認
   ! grep -RIn 'type *= *"MOCK"' infra/ \
     || { echo "[FAIL] MOCK integration found" >&2; exit 1; }
   ! grep -RIn 'method.response.header.Access-Control' infra/ \
     || { echo "[FAIL] response_parameters CORS declaration found (RD-011 違反)" >&2; exit 1; }
   ```
2. **本タスク内**で上記 RED スクリプトを `bash tests/infra/test-frontend-plan.sh` で実行し、リソース未追加状態で **必ず exit 1** すること（RP-010: terraform validate だけの always-pass を禁止）
3. （連動 RED）task09 の `OPTIONS /todos` IntegrationTest を **本タスクの RED 完了条件として併記** し、Lambda + Terraform 反映前は失敗することを確認

### GREEN
4. `infra/main.tf` (もしくは新規 `infra/frontend.tf`) に上記リソースを追加
5. `infra/outputs.tf` に `frontend_bucket` / `frontend_url` を追加
6. `terraform -chdir=infra fmt` / `terraform validate` が成功
7. floci 起動状態で `terraform -chdir=infra plan` がエラー無し
8. `terraform apply` 後に `curl -i -X OPTIONS http://<endpoint>/restapis/.../dev/_user_request_/todos` で `HTTP/1.1 204` + CORS ヘッダを確認 (task02-01 と組み合わせて)

### REFACTOR
9. AWS_PROXY 透過に依拠する旨を Terraform コメントに明記
10. `frontend_url` の用途コメント (nginx ベース URL の組み立てヒント) を追記
11. `aws_s3_bucket.frontend` には実 AWS 流出防止のため tags + コメント (floci ローカル限定) を追加
