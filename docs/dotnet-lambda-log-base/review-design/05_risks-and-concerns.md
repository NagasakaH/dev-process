# リスク・懸念事項

## 調査結果で指摘されたリスクへの対応

| リスク | 設計での対応 | 対応状況 |
|--------|-------------|----------|
| Lambda タイムアウト時のログロスト | try-finally での FlushAsync 呼び出し | ✅ 対応済み |
| PutLogEvents API スロットリング | バッチ送信でAPI呼び出し削減 | ✅ 対応済み |
| Delivery Class 制約 | 異常系は Standard Class に送信 | ✅ 対応済み |

## 設計上の追加リスク

| リスク | 重大度 | 対策 |
|--------|--------|------|
| Lambda タイムアウトで FlushAsync が完了しない場合 | 🟡 Minor | バッファサイズを小さく保つことで軽減。完全な解決は困難（Lambda の制約）|
| 大量ログでバッチサイズ（1MB）を超える場合 | 🟡 Minor | SendAsync 内でバッチ分割ロジックを実装する必要あり |

## 判定

✅ **重大なリスクなし。Minor リスク2件は実装フェーズで対応可能。**
