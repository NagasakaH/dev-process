## 概要

floci を AWS シミュレータとして利用し、API Gateway、.NET 8 Lambda、Step Functions を組み合わせた Todo API サンプルアプリを作成する。Terraform でローカル/手動デプロイできる IaC を用意し、GitLab CI では xUnit の単体テスト、Lambda テスト、API Gateway 経由の E2E テストまでを floci 上で実行できる仕組みを構築する。

---

## 目的

.NET Lambda と AWS サーバーレス構成を floci で継続的に検証できる参照実装を提供し、Terraform デプロイ、GitLab CI、テスト、デバッグの標準的な手順を README から再現できるようにする。

---

## 背景

https://github.com/floci-io/floci.git を関連リポジトリとして参照し、floci が提供する AWS シミュレータ機能を使って API Gateway/Lambda/Step Functions の動作検証を CI 内で完結させる。実AWSへのデプロイはユーザーが手動で実行する前提とし、CI の責務は floci 上の検証までとする。

---

## 要件

### 機能要件

- Todo API のサンプルを作成する。CRUD を中心とし、Step Functions は簡易的な Todo 処理フローで利用する。
- .NET 8 Lambda を実装し、API Gateway から呼び出せる構成にする。
- Step Functions ステートマシンを Terraform で定義し、Lambda と連携させる。
- Terraform で API Gateway、Lambda、Step Functions、および必要な権限を定義する。
- GitLab CI で xUnit の単体テストを実行する。
- GitLab CI で Lambda ハンドラ/関数レベルのテストを実行する。
- GitLab CI で floci を使い、API Gateway 経由の E2E テストを実行する。
- README にローカル実行、CI、Terraform デプロイ、テスト、デバッグ方法を記載する。

### 非機能要件

- サンプルとして読みやすく、最小構成で再現しやすい構成にする。
- GitLab CI 上で floci による検証を自動化し、実AWS資格情報なしでも主要テストを実行できるようにする。
- Terraform state はローカル tfstate を基本とし、GitLab managed Terraform state 化は README の補足として記載する。
- .NET 8 と Terraform の一般的な開発フローに沿った構成にする。
