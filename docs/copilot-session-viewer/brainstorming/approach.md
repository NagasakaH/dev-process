# Brainstorming アプローチ

## 採用アプローチ: ハイブリッドコンテナ化

### 概要

copilot-session-viewer を自己完結型コンテナとして動作させるために、1コンテナに Next.js (viewer) + tmux + Copilot CLI を同居させるハイブリッドアプローチを採用する。

### 背景と動機

現在の構成では、dev-process コンテナから別コンテナを管理するアプローチを取っているが、tmux が予期せず終了する問題が発生している。中間ステップとして、ターゲットプロジェクト自体を自己完結型のローカル/コンテナ専用モードで動作させることを目指す。

### アーキテクチャ

```
┌─────────────────────────────────────────────┐
│  Container (compose.yaml)                   │
│                                             │
│  ┌──────────┐  ┌──────┐  ┌──────────────┐  │
│  │ Next.js  │  │ tmux │  │ Copilot CLI  │  │
│  │ (viewer) │  │      │  │ (cplt)       │  │
│  └──────────┘  └──────┘  └──────────────┘  │
│                                             │
│  ┌──────────────────────────────────────┐   │
│  │ tini (PID 1 init)                   │   │
│  └──────────────────────────────────────┘   │
│                                             │
│  Volumes:                                   │
│  - $HOME/.copilot (isolated)                │
│  - .env (PAT, BASIC_AUTH)                   │
└─────────────────────────────────────────────┘
```

### 主要コンポーネント

#### 1. Dockerfile

- ベースイメージ: Node.js LTS (Debian ベース)
- インストール: tmux, tini, Copilot CLI
- dev-process の `start-tmux.sh` パターンを参考にエントリポイントスクリプトを作成
- tini を PID 1 として使用し、シグナルハンドリングとゾンビプロセス回収を適切に行う

#### 2. compose.yaml

- 単一サービス構成
- `.env` からの環境変数注入
- ボリュームマウント設定
- ヘルスチェック定義

#### 3. セッション検出のローカル専用モード

- 環境変数 `DISABLE_DOCKER_DETECTION` でホスト Docker 検出を無効化
- ローカル tmux 検出は常に有効
- コンテナ内ではローカルプロセスとして tmux を直接管理

### 根拠

#### なぜ 1 コンテナ構成か

- **シンプルさ**: プロセス間通信やコンテナ間連携の複雑さを排除
- **tmux の安定性**: ホスト-コンテナ間の Docker exec 経由ではなく、ローカルプロセスとして tmux を管理することで安定性を向上
- **dev-process パターンとの整合性**: 既存の `start-tmux.sh` / tini パターンを流用可能

#### なぜ Docker 検出を無効化可能にするか

- コンテナ内では Docker ソケットへのアクセスが不要・不適切
- ローカル tmux 検出のみで十分な機能を提供
- フラグ制御により将来の要件変更に対応可能

#### なぜ Vitest + Playwright か

- **Vitest**: Next.js エコシステムとの親和性、Vite ベースの高速実行
- **Playwright**: ブラウザベースの E2E テストに加え、コンテナ内実行のサポートが充実
- 両ツールとも TypeScript ファーストで copilot-session-viewer のスタックと整合

### テスト戦略

| レイヤー    | ツール     | 対象                                               |
| ----------- | ---------- | -------------------------------------------------- |
| Unit        | Vitest     | セッション検出ロジック、環境変数処理、認証設定注入  |
| Integration | Vitest     | tmux 検出 + セッション管理統合、.env 設定パイプライン |
| E2E         | Playwright | コンテナ起動→viewer起動→tmux確認→認証確認フロー   |

### リスクと対策

| リスク                                  | 対策                                              |
| --------------------------------------- | ------------------------------------------------- |
| tini + tmux の組み合わせでシグナル問題  | dev-process の既存パターンを参考に事前検証         |
| Playwright コンテナ内実行の環境依存問題 | Playwright 公式 Docker イメージを参考にセットアップ |
| 既存機能へのリグレッション              | Unit/Integration テストで既存ロジックをカバー      |
