## 作業内容

### 目的

xterm.js ラッパーコンポーネント `TerminalView` とモーダル UI `TerminalModal` を実装する。TerminalView は WS 接続管理 + ターミナル描画の責務を持ち（MRD-012）、TerminalModal は UI 表示のみを担当する。

### 設計参照

- `02_interface-api-design.md` §4 コンポーネントインターフェース
- `03_data-structure-design.md` §3.2 接続ライフサイクル（状態遷移図）
- MRD-012: コンポーネント責務明確化

### 実装ステップ

1. **テスト作成（RED）**
   - `src/components/__tests__/TerminalView.test.tsx`: UT-26, UT-27, UT-28
   - `src/components/__tests__/TerminalModal.test.tsx`: UT-21, UT-22, UT-23, UT-24, UT-25

2. **TerminalView 実装（GREEN）**
   - `useRef` で DOM コンテナ管理
   - `useEffect` で xterm.js Terminal 初期化・FitAddon ロード・`open()`
   - `useTerminalWebSocket` フックを内部で使用
   - `onOutput` → `terminal.write(data)`
   - `onData` → `sendInput(data)`
   - `FitAddon.fit()` 後に `sendResize(cols, rows)`
   - ResizeObserver でコンテナサイズ変更を監視
   - next-themes の `useTheme()` でテーマ同期
   - unmount 時: `terminal.dispose()`, `disconnect()`

3. **TerminalModal 実装（GREEN）**
   - `fixed inset-0 z-50` のオーバーレイ
   - バックドロップ（半透明背景）クリックで閉じない（誤操作防止）
   - ✕ボタン + Escape キーで `onClose`
   - ヘッダー: セッション情報（sessionId, summary）
   - ボディ: `<TerminalView sessionId={session.id} />`
   - エラー/切断状態の表示

4. **DisconnectedMessage コンポーネント**（TerminalView 内）
   - connectionState が "disconnected" / "error" の場合に表示
   - 理由に応じたメッセージ

5. **REFACTOR**

```typescript
// src/components/__tests__/TerminalModal.test.tsx

import { render, screen, fireEvent, cleanup } from "@testing-library/react";
import { describe, it, expect, vi, afterEach } from "vitest";
import { TerminalModal } from "../TerminalModal";
import type { ActiveSession } from "../../lib/terminal";

const mockSession: ActiveSession = {
  id: "session-001",
  sessionState: "working",
  lastActivity: new Date().toISOString(),
  tmuxPane: "0:1.0",
  summary: "Test session",
};

vi.mock("../TerminalView", () => ({
  TerminalView: vi.fn(() => <div data-testid="terminal-view" />),
}));

describe("TerminalModal", () => {
  const mockOnClose = vi.fn();
  afterEach(() => { cleanup(); mockOnClose.mockClear(); });

  // UT-21: モーダルレンダリング
  it("モーダルが fixed + z-50 で表示される", () => {
    render(<TerminalModal session={mockSession} onClose={mockOnClose} />);
    const modal = screen.getByRole("dialog");
    expect(modal).toBeInTheDocument();
    expect(modal.className).toContain("fixed");
    expect(modal.className).toContain("z-50");
  });

  // UT-22: ✕ボタン
  it("✕ボタンで onClose が呼ばれる", () => {
    render(<TerminalModal session={mockSession} onClose={mockOnClose} />);
    const closeBtn = screen.getByRole("button", { name: /閉じる|close/i });
    fireEvent.click(closeBtn);
    expect(mockOnClose).toHaveBeenCalledTimes(1);
  });

  // UT-23: Escape キー
  it("Escape キーで onClose が呼ばれる", () => {
    render(<TerminalModal session={mockSession} onClose={mockOnClose} />);
    fireEvent.keyDown(document, { key: "Escape" });
    expect(mockOnClose).toHaveBeenCalledTimes(1);
  });

  // UT-24: セッション情報表示
  it("sessionId と summary が表示される", () => {
    render(<TerminalModal session={mockSession} onClose={mockOnClose} />);
    expect(screen.getByText(/session-001/)).toBeInTheDocument();
    expect(screen.getByText(/Test session/)).toBeInTheDocument();
  });

  // UT-25: エラー状態
  it("エラー状態で適切なメッセージが表示される", () => {
    // connectionState が "error" の場合
    // TerminalView 内で表示される → TerminalView のモックを error 状態に
  });
});
```
